Angular 元素拖拽效果指令的实现及优化方案

Posted on 2020-04-04

1. requestAnimationFrame 的优势

  1. 防止丢帧: 由系统来决定回调函数的执行时机，它能保证回调函数在屏幕每一次的刷新间隔中只被执行一次，这样就不会引起丢帧现象，也不会导致动画出现卡顿的问题。
  2. CPU 节能: 使用 setTimeout 实现的动画，当页面被隐藏或最小化时，setTimeout 仍然在后台执行动画任务，由于此时页面处于不可见或不可用状态，刷新动画是没有意义的，完全是浪费 CPU 资源。而 requestAnimationFrame 则完全不同，当页面处理未激活的状态下，该页面的屏幕刷新任务也会被系统暂停，因此跟着系统步伐走的 requestAnimationFrame 也会停止渲染，当页面被激活时，动画就从上次停留的地方继续执行，有效节省了 CPU 开销。
  3. 函数节流: 在高频率事件(resize,scroll 等)中，为了防止在一个刷新间隔内发生多次函数执行，使用 requestAnimationFrame 可保证每个刷新间隔内，函数只被执行一次，这样既能保证流畅性，也能更好的节省函数执行的开销。一个刷新间隔内函数执行多次时没有意义的，因为显示器每 16.7ms 刷新一次，多次绘制并不会在屏幕上体现出来。

2. 为什么使用 Angular 指令

在 Angular 中有三种类型的指令：

  1. 组件 — 拥有模板的指令

  2. 结构型指令 — 通过添加和移除 DOM 元素改变 DOM 布局的指令

  3. 属性型指令 — 改变元素、组件或其它指令的外观和行为的指令。

结构型指令修改视图的结构。例如，NgFor 和 NgIf。

属性型指令改变一个元素的外观或行为。例如，内置的 NgStyle 指令可以同时修改元素的多个样式。

结合以上的指令特性，写出一种只要添加上该指令的元素，该元素就拥有了可拖拽的功能。

3. 拖拽优化方案

优化目标：

  1. 最大可能性的减少浏览器的重排和重绘。
  2. 使用NgZone runOutsideAngular避免拖拽的时候angular跟踪变化，导致不必要的检测。

实现方案：

  1. 点击该元素的时候，使得该元素 visibility:hidden,在 clone 一份新的元素，使得该元素的 position:absolute 或者是 fixed，这样元素虽然不可见，但还是占据着位置，不会导致浏览器进行重排，clone 的元素布局为 absolute，脱离了当前的文档流，这样 clone 的新元素怎么拖拽也不会影响文档流。
  2. 当 clone 元素拖拽过程中，去判断和其它兄弟元素的位置，通过 translate3d 代替 left 或 top 去改变其它兄弟元素的位置（这样可以启动 GPU 加速渲染，从而避免重排），直到一次拖拽结束之后，把相互变化的两个元素的索引 emit 出去。
  3. 订阅 emit 事件的函数通过索引来更新原有的数据模型的顺序，从而实现元素的重新排列。
